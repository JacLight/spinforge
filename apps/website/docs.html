Quick Start

Deploy your first app on SpinForge in under 60 seconds.

1 Install the CLI

npm install -g @spinforge/cli

2 Authenticate

spinforge login

This will open your browser to complete authentication.

3 Deploy

spinforge deploy

Run this in your project directory. SpinForge will automatically detect your framework.

🎉 That's it!

Your app is now live on SpinForge's global edge network. The CLI will output your deployment URL.

Installation

Using npm

npm install -g @spinforge/cli

Using yarn

yarn global add @spinforge/cli

Using pnpm

pnpm add -g @spinforge/cli

Verify Installation

spinforge --version

Your First Deploy

Let's deploy a simple static website to see SpinForge in action.

1. Create a simple HTML file

<!DOCTYPE html>
<html>
<head>
    <title>Hello SpinForge!</title>
</head>
<body>
    <h1>Welcome to SpinForge! 🚀</h1>
    <p>This site is running on the edge.</p>
</body>
</html>

2. Deploy it

spinforge deploy

3. View your site

SpinForge will give you a URL like https://your-app-abc123.spinforge.io

Deploying Static Sites

Static sites are the simplest to deploy. Just point SpinForge at your build directory.

# Build your static site first
npm run build
# Deploy the build directory
spinforge deploy ./dist

Tip: SpinForge automatically serves index.html for directory requests and handles 404s gracefully.

Deploying Next.js

SpinForge automatically detects Next.js projects and optimizes them for edge deployment.

# In your Next.js project root
spinforge deploy

Automatic Optimizations

  • Image optimization with next/image
  • API routes deployed as edge functions
  • Automatic ISR (Incremental Static Regeneration)

Custom Domains

Add your own domain to your SpinForge deployment.

Add a domain

spinforge domains add example.com

Configure DNS

Add a CNAME record pointing to your SpinForge deployment:

CNAME @ your-app.spinforge.io

SSL is automatic! SpinForge provisions and renews SSL certificates automatically.
